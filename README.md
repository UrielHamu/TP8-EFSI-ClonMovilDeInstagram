
Este proyecto fue realizado utilizando ChatGPT como herramienta de apoyo durante el desarrollo, como se puede ver a lo largo de esta conversación: https://chatgpt.com/share/6a4e6951-4148-83e9-b6d9-af7c3ff48497. Desde el principio, usamos la IA para ordenar mejor la consigna, entender qué se nos pedía y transformar una idea general en un prompt concreto para arrancar, trabajamos con React Native, JS y Expo Go.

El objetivo del trabajo era crear un clon móvil de Instagram, pasando de lo que ya conocíamos en React web a una aplicación mobile. Para eso, le pedimos a ChatGPT que actuara como un desarrollador senior, pero que explicara todo de manera clara porque todavía estamos aprendiendo React Native. La IA nos ayudó en como manejar de carpetas dentro de src, separando pantallas, componentes, navegación, servicios y datos simulados. Esto nos permitió trabajar de una forma más ordenada y entender mejor qué responsabilidad tenía cada archivo.

Durante el desarrollo no copiamos todo sin revisar, sino que fuimos probando cada parte en el proyecto. Por ejemplo, implementamos el feed principal, conectamos la app a una API externa de imágenes de gatos, armamos componentes reutilizables como los posteos y la barra de acciones, y configuramos la navegación entre el feed, el detalle de publicación y el perfil. Cuando algo no funcionaba, como errores con Expo, rutas de imágenes o dependencias, fuimos consultando nuevamente a la IA o a la profesora y corrigiendo el código paso a paso.

También aparecieron decisiones de diseño. Al principio, el perfil usaba un símbolo que no quedaba como queríamos, entonces fuimos modificando la vista hasta lograr que el avatar principal fuera una foto de persona y que abajo quedaran los íconos de texto del perfil. Además, nos dimos cuenta de que las publicaciones del perfil no podían quedar estáticas, por lo que agregamos la posibilidad de tocarlas y navegar al detalle de cada publicación, haciendo que la app se parezca más al funcionamiento real de Instagram.

Una parte importante del proceso fue entender que la IA no siempre da una respuesta perfecta desde el primer intento. En algunos momentos hubo errores o confusiones, como rutas incorrectas de archivos, diferencias entre .png y .jpg, o problemas de versiones con Expo. También aprendimos que hay decisiones que pueden hacerse de más de una manera, como dónde ubicar los estilos. Aunque ChatGPT propuso usar StyleSheet.create() dentro de cada archivo, entendimos que también podrían separarse los estilos en archivos aparte. Finalmente decidimos dejarlos dentro de los .jsx porque para este trabajo era más claro visualmente.

En conclusión, ChatGPT fue una herramienta útil para avanzar, entender errores y organizar el proyecto, pero no reemplazó nuestro trabajo. Tuvimos que leer, probar, corregir, tomar decisiones y adaptar las respuestas a lo que realmente necesitábamos. Este TP nos sirvió para aprender mejor cómo funciona React Native, cómo se estructura una app mobile con Expo y por qué es importante revisar críticamente lo que propone una IA.
